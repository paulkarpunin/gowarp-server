# WARP Manager

Bash-скрипт для интеграции [Cloudflare WARP](https://1.1.1.1/) с VPN-серверами на базе **3X-UI** (Xray) и **AmneziaWG** (WireGuard in Docker). Управление через интерактивное SSH-меню или Telegram-бот.

## Возможности

- Автоопределение окружения (3X-UI / AmneziaWG / оба режима)
- Установка и настройка Cloudflare WARP в один клик
- Перевыпуск WARP-ключа без ручного вмешательства
- Выборочная маршрутизация клиентов через WARP (AmneziaWG)
- Telegram-бот с inline-keyboard для управления с телефона
- Персистентность настроек после перезапуска контейнера
- Автоустановка команды `gowarp` в `/usr/local/bin`

| Другие дистрибутивы | ⚠️ только AmneziaWG-режим |

## Режимы работы

### Режим 3X-UI

Cloudflare WARP устанавливается нативно на сервер и работает как SOCKS5-прокси. Xray направляет через него нужные домены.

```
Клиент → Xray (3X-UI) → SOCKS5 127.0.0.1:40000 → WARP → Cloudflare → Интернет
```

**Что настраивается автоматически:**
1. Устанавливается `cloudflare-warp` из официального репозитория Cloudflare
2. WARP регистрируется и переводится в режим SOCKS5-прокси
3. Выводится готовый JSON для вставки в настройки Xray (Outbound + Routing)

**JSON Outbound для 3X-UI:**
```json
{
  "tag": "warp",
  "protocol": "socks",
  "settings": {
    "servers": [
      { "address": "127.0.0.1", "port": 40000 }
    ]
  }
}
```

**Пример Routing Rule (только выбранные сайты через WARP):**
```json
{
  "outboundTag": "warp",
  "domain": [
    "geosite:openai",
    "geosite:netflix",
    "geosite:disney",
    "geosite:spotify",
    "domain:chat.openai.com",
    "domain:claude.ai"
  ]
}
```

### Режим AmneziaWG

WARP разворачивается внутри Docker-контейнера AmneziaWG как WireGuard-интерфейс `warp`. Маршрутизация настраивается per-client через `ip rule` и `iptables MASQUERADE`.

```
Клиент → AmneziaWG Docker → warp (WireGuard) → Cloudflare → Интернет
```

**Что настраивается автоматически:**
1. Скачивается [`wgcf`](https://github.com/ViRb3/wgcf) и регистрируется WARP-аккаунт
2. Генерируется WireGuard-профиль и разворачивается интерфейс `warp` внутри контейнера
3. Настраивается `ip rule` + `iptables` для per-client маршрутизации
4. Блок запуска встраивается в `start.sh` контейнера для автовосстановления после рестарта

**Управление клиентами:** через пункт меню «Управление клиентами WARP» можно для каждого VPN-пользователя включить или выключить туннелирование через WARP.

## Telegram-бот

Бот позволяет управлять WARP с телефона без SSH-доступа.

**Настройка:**
1. Создайте бота через [@BotFather](https://t.me/BotFather) → `/newbot` → скопируйте токен
2. В меню скрипта выберите пункт `8) Настройка и управление ботом`
3. Вставьте токен, отправьте боту любое сообщение для определения Chat ID
4. Запустите бота — он регистрируется как systemd-сервис `warp-bot`

**Доступные команды бота:**

| Кнопка | Действие |
|--------|----------|
| 📊 Статус | Статус WARP, IP-адреса, инфо о контейнере |
| 🌐 IP | Реальный IP сервера и IP через WARP |
| ▶️ Запустить | Подключить WARP |
| ⏹ Остановить | Отключить WARP |
| 🔑 Перевыпуск | Сменить WARP-ключ (новая регистрация) |
| 👥 Клиенты WARP | Включить/выключить WARP для каждого клиента |
| 🔄 Контейнер | Перезапустить Docker-контейнер AmneziaWG |
| 💻 Система | CPU, RAM, диск, uptime |

Управление ботом через systemd:
```bash
systemctl status warp-bot
systemctl restart warp-bot
journalctl -u warp-bot -f
```

## Конфигурация

Настройки хранятся в `/etc/warp-manager/config`:

```bash
SOCKS_PORT="40000"    # Порт SOCKS5-прокси (только для 3X-UI)
BOT_TOKEN=""          # Токен Telegram-бота
BOT_CHAT_ID=""        # Chat ID для ограничения доступа к боту
MODE=""               # Режим: 3xui | amnezia | both
CONTAINER=""          # Имя Docker-контейнера AmneziaWG
```

## Удаление

Полное удаление через меню: пункт `10) Полное удаление`.

Вручную:
```bash
gowarp
# выбрать пункт 10
```

Что удаляется:
- `cloudflare-warp` и его репозиторий (режим 3X-UI)
- WireGuard-интерфейс `warp` и файлы в контейнере (режим AmneziaWG)
- Systemd-сервис `warp-bot`
- Команда `/usr/local/bin/gowarp`
- Конфиг `/etc/warp-manager/` и лог `/var/log/warp-manager.log`
- `wgcf` и WARP-профили из `/root/`
